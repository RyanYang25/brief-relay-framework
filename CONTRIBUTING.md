# 贡献指南 (CONTRIBUTING)

感谢你关注「简报接力框架 BRF」！欢迎通过 Issue 和 Pull Request 参与改进。

## 如何贡献

1. Fork 本仓库到你的账号；
2. 创建特性分支：`git checkout -b feature/your-idea`；
3. 提交改动：`git commit -m "feat: ..."`；
4. 推送并提 PR：`git push origin feature/your-idea` → 在 GitHub 发起 Pull Request。

## 提交规范

- 方法规范（`简报接力-v1.0.md`）的修改请保持中文、结构稳定；重大调整请在 PR 描述中说明动机。
- 模板（`template/`）的改动请确保**中英文模板保持同步**——改了一侧要同步另一侧。
- 新增文件请遵循仓库已有的目录与命名习惯，不另起方案。

## 议题（Issue）

- 发现问题或想讨论想法，直接开 Issue。
- 涉及方法理念的讨论，建议先在 Issue 中说明背景，便于他人理解。

## 行为准则

- 友善、就事论事；尊重不同项目场景下的实践差异。
