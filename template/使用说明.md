# 模板使用说明

本目录提供「简报接力框架 BRF」的**开箱即用模板**。把它复制到你的任意项目根目录，按占位符填写即可，无需先读懂完整方法规范。

## 一分钟上手

1. 把 `template/` 整个目录复制到你的项目根目录（或只取里面的文件）：

   ```
   your-project/
   ├── BRIEF.md
   ├── project-context/
   │   ├── context/
   │   │   ├── current.md
   │   │   └── standards.md
   │   ├── references/
   │   ├── handoffs/
   │   ├── .working/
   │   └── changelog.md
   └── {会议记录目录}/
   ```

2. 打开 `BRIEF.md`，把 `{项目名称}`、`{项目描述}` 等占位符替换为你的真实内容。
3. 把你的会议记录目录名填到 `{会议记录目录}`（默认建议 `B02 会议记录/` 或你自己的编码）。
4. 遇到状态更新时，按 `BRIEF.md` 底部「更新 SOP」的指示联动改文件，**不要只改 BRIEF.md**。

## 文件职责

- `BRIEF.md`：唯一入口 + 演进引擎。新 Agent / 换平台时只读它即可接手。
- `context/current.md`：当前状态、进行中任务、近期决策（含依据/理由）。更新最频繁。
- `context/standards.md`：输出规范、协作流程、历史踩坑。只增不改。

## 三件套（接力的保证）

- **@context 契约头**：每个文件顶部的 HTML 注释，声明角色与接力关系（谁产、谁消费、下一步动谁）。
- **接力棒（downstream）**：契约头里的 `downstream` 字段，告诉 Agent 改完本文件下一步该动谁。
- **会话恢复协议**：换平台 / 中断恢复时，先读 BRIEF + current 比对时间再干活，不丢上下文。
